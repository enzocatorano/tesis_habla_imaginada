TRABAJO REALIZADO:

1. Exploración del repositorio GitHub externo LTU-Machine-Learning/Rethinking-Methods-Inner-Speech:
   - Repositorio del paper sobre decodificación de habla interna (inner speech)
   - Dataset SINC (mismo que usamos nosotros) - Universidad Nacional del Litoral (Argentina)
   - 15 sujetos, 6 canales EEG, 128 Hz (downsampled desde 1024 Hz)
   - Tareas: vocales (5 clases) y palabras/comandos (6 clases)

2. Análisis comparativo de arquitecturas CNN:
   - iSpeech-CNN (LTU): ~760K parámetros (6 capas conv, separable temporal/espacial)
   - EEGNet: ~7K parámetros (compacta, con sesgo inductivo para EEG)
   - ShallowConvNet: ~18K parámetros (basada en FBCSP)
   - DeepConvNet: ~120K parámetros (4 bloques profundos)
   - Las arquitecturas anteriores (EEGNet, Shallow, Deep) no superaron la probabilidad base

3. Implementación de iSpeech-CNN en src/models.py:
   - Nueva clase iSpeechCNN basada en la arquitectura del repositorio LTU
   - 6 capas convolucionales con separación temporal/espacial explícita
   - Parámetros por defecto: F1=20 (vocales), dropout=0.0002
   - Arquitectura: Conv1a[1×5] temporal → Conv1b[6×1] espacial → ... → FC
   - Total ~711K parámetros para 5 clases

4. Modificación de src/experimento_CNNs.py:
   - Agregado import de iSpeechCNN
   - Agregado bloque de selección de modelo para iSpeechCNN

5. Preparación de prueba preliminar:
   - Configuración para ejecutar con 1 sujeto, sin augmentación
   - Solo datos submuestreados (128 Hz), sin segmentado de ventana
   - Hiperparámetros del paper original: LR=0.001, epochs=60, patience=30

OBJETIVO: Probar si el modelo más grande (iSpeech-CNN) puede resolver el problema
de clasificación de habla imaginada que las redes más pequeñas no lograron resolver.

TRABAJO REALIZADO:

1. ANÁLISIS DEL FILTRO FIR DE LIWICKI (BUG DESCUBIERTO):
   - Revisamos el código de preprocesamiento de Liwicki (prepro_DFICA_for_vowels.m)
   - Parámetros del filtro: lowpass n=372 @ 2Hz, highpass n=1204 @ 40Hz
   - Descubrimiento: el filtro FIR de orden 372 tiene una zona de transición RIDÍCULAMENTE
     amplia. A 10 Hz ya pierde -58dB, destruyendo casi toda la señal EEG útil.
   - Resultado: El filtro elimina ~100% de la potencia en 2-10 Hz y 10-20 Hz, solo pasa 20-40 Hz
   - Los datos preprocesados de Liwicki tienen el filtro ya aplicado (no hay versión sin filtro)

2. ANÁLISIS DE NUESTROS DATOS PREPROCESADOS:
   - Revisamos data/preprocessed/ para todos los sujetos
   - Formato de labels: [modalidad, estímulo, artefacto]
     - Modalidad: 1=Imaginada, 2=Pronunciada
     - Estímulo: 1-5 (vocales), 6-11 (comandos)
     - Artefacto: 1=Limpio, 2=Parpadeo

3. CREACIÓN DEL DATASET LIMPIO (data/clean_preprocessed/):
   - Filtramos data/preprocessed/ por:
     * Solo imaginada (modalidad == 1)
     * Solo limpia (artefacto == 1, sin parpadeo)
   - Resultado: 5,449 trials (de ~9,400 originales, 58% retención)
   - Mejor sujeto: S09 con 428 trials limpios
   - Archivos: S01_clean.npz hasta S15_clean.npz

4. CREACIÓN DEL DATASET SINTÉTICO CON LABELS INCRUSTADAS (data/synthetic_labels/):
   - Propósito: Testing del pipeline de entrenamiento
   - Método: Añadir sinusoidal a frecuencia única para cada estímulo
   - Frecuencias: 20Hz a 40Hz (de 2 en 2 Hz), magnitud FFT objetivo = 10000
   - Formula: señal += 39.0625 * sin(2*pi*freq*t)
   - Verificación: FFT muestra picos exactos en las frecuencias设计adas
   - Archivos: S01_synthetic.npz hasta S15_synthetic.npz (9,315 trials)
